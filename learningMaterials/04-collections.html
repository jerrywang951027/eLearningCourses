Notes

Page 1

Collections manage groups of values. Lists are ordered and mutable, making them useful for tasks, scores, rows, names, and other sequences that may change. They support indexing, slicing, appending, inserting, removing, sorting, and iteration.

Tuples are ordered and immutable. They fit fixed records such as coordinates, RGB values, and multiple return values. Tuple unpacking assigns positions to names, as in x, y = point.

Dictionaries map keys to values. They are natural for labeled data such as settings, user profiles, inventory counts, and lookup tables. Access with data[key] when the key must exist; use get(key, default) when absence is expected.

Sets store unique values and are excellent for membership tests, duplicate removal, union, intersection, and difference operations.

Page 2

Choose collections by meaning. Use a list when order and change matter. Use a tuple when a fixed positional group matters. Use a dictionary when labels map to values. Use a set when uniqueness or fast membership matters.

Many real structures combine collections: a list of dictionaries for table rows, a dictionary of lists for grouping, or a set of seen IDs while scanning records.

Loop directly over values when indexes are unnecessary. Use enumerate() when you need index and value. Use dict.items() when you need both keys and values.

Avoid modifying a list while iterating over it unless you are iterating over a copy or building a new collection. Mutation during iteration can skip items and create subtle bugs.

Small Coding Challenges

1. Unique Words

Return sorted unique lowercase words.

Reference implementation:

def unique_words(text):
    return sorted(set(text.lower().split()))

2. Inventory Total

Sum all counts in an inventory dict.

Reference implementation:

def inventory_total(inventory):
    total = 0
    for count in inventory.values():
        total += count
    return total

3. Top Score

Return largest score or None for empty.

Reference implementation:

def top_score(scores):
    if not scores:
        return None
    return max(scores)

4. Email Lookup

Return email by username or missing.

Reference implementation:

def lookup_email(users, username):
    return users.get(username, "missing")

5. Format Point

Unpack (x, y) and return a labelled string.

Reference implementation:

def format_point(point):
    x, y = point
    return f"(x={x}, y={y})"
