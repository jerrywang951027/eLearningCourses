Notes

Page 1

Modules keep code organized. A module is a Python file that can be imported. The standard library provides modules for math, paths, JSON, command-line arguments, dates, random values, and more. Your own files can also become modules when you separate reusable logic from scripts.

Virtual environments isolate dependencies. Create one with python -m venv .venv, activate it, and install packages with pip. Recording dependencies lets another person reproduce your project.

File operations persist data. Use with open(...) or pathlib methods so files are closed properly. Prefer structured formats like JSON or CSV when data needs to be read back reliably.

Command-line programs receive input through arguments. sys.argv is the raw list; argparse is better for real tools because it defines options, help text, types, defaults, and validation.

Page 2

Object-oriented programming groups state and behavior. A class is a blueprint; an object is an instance. __init__ initializes state, self refers to the current instance, attributes store data, and methods define behavior.

Exceptions communicate failure. Catch specific exceptions such as ValueError, KeyError, and FileNotFoundError. Raise exceptions when a function cannot honestly complete its contract.

Projects combine all course pieces: data structures, functions, loops, modules, files, classes, and error handling. Separate core logic from input/output so the logic can be tested without a user sitting at the keyboard.

Build the smallest working version first, then add features. When code becomes hard to change, look for repeated blocks, hidden global state, and functions or classes with too many responsibilities.

Small Coding Challenges

1. Read Lines

Return stripped non-empty lines from a file.

Reference implementation:

from pathlib import Path

def read_nonempty_lines(path):
    return [line.strip() for line in Path(path).read_text().splitlines() if line.strip()]

2. Write JSON

Write a dictionary to JSON with indentation.

Reference implementation:

import json
from pathlib import Path

def write_json(path, data):
    Path(path).write_text(json.dumps(data, indent=2))

3. Bank Account

Create a class with deposit and withdraw validation.

Reference implementation:

class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.balance = balance

    def deposit(self, amount):
        if amount <= 0:
            raise ValueError("deposit amount must be positive")
        self.balance += amount

    def withdraw(self, amount):
        if amount <= 0:
            raise ValueError("withdraw amount must be positive")
        if amount > self.balance:
            raise ValueError("insufficient funds")
        self.balance -= amount

4. Safe Integer

Convert text to int, returning None for invalid values.

Reference implementation:

def safe_int(text):
    try:
        return int(text)
    except ValueError:
        return None

5. Project Boundary

Separate parsing from adding logic.

Reference implementation:

def add(a, b):
    return a + b

def parse_and_add(left_text, right_text):
    return add(int(left_text), int(right_text))
