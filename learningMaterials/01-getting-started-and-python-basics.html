Notes

Page 1

A beginner Python workflow starts with installing Python, choosing an editor, creating a .py file, running it from the terminal, reading the output, and changing the code in small steps. The terminal matters because it is where scripts run, packages install, and errors show their most useful details.

Python executes a script from top to bottom unless control flow changes the path. Comments begin with # and should explain intent, not restate obvious code. The print() function displays values, while variables give names to values so a program can reuse and transform information.

Python is dynamically typed: names do not declare a permanent type, but every value has a runtime type. Important beginner values include strings, integers, floats, booleans, and None. The type() function is useful when you are unsure what kind of value you have.

Tracebacks are maps. The final line usually names the error type and message, and the lines above point to the file and line involved. Syntax errors mean Python cannot parse the program. Logic errors mean the program runs but does the wrong thing.

Page 2

Good Python style starts with clear names, consistent indentation, and small experiments. Use snake_case for variables and functions, reserve short names for tiny local contexts, and avoid shadowing built-ins such as list, str, or type.

Indentation defines blocks in Python. Code beneath if, for, while, def, and class must be consistently indented, conventionally with four spaces. Mixed or missing indentation changes meaning or raises errors.

Beginners often move from literals to variables: print(2 + 3) becomes left = 2, right = 3, print(left + right). That shift is important because it makes programs adaptable and easier to read.

The strongest early habit is prediction. Before running code, say what value or output you expect. When reality differs, the gap points directly to the concept you need to revisit.

Small Coding Challenges

1. Greeting Function

Write greet(name) that returns Hello, <name>!.

Reference implementation:

def greet(name):
    return f"Hello, {name}!"

2. Type Name

Write type_name(value) that returns the runtime type name.

Reference implementation:

def type_name(value):
    return type(value).__name__

3. Full Name

Combine first and last names with a space.

Reference implementation:

def full_name(first, last):
    return first + " " + last

4. Receipt Total

Return subtotal plus tax by rate.

Reference implementation:

def total_with_tax(subtotal, tax_rate):
    return subtotal + subtotal * tax_rate

5. Debug Message

Return a labelled value such as age=31.

Reference implementation:

def debug_value(name, value):
    return f"{name}={value}"
